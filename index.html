Synchronous.. Asynchronous.. ?

Latency impact can be reduced with asynchronous processing

Mashup service example

Return rich product information by retrieving data from multiple http services (product, price)

GET /productdetails/{productId}

Synchronous implementation

GET /productdetails/{productId}


val product: Product = productClient.getProduct(productId)
val price: Price = priceClient.getPrice(productId)

val productDetails = ProductDetails(product, price)
val json = toJson(productDetails)

// ...and then return the http response
    - Latency as a hidden effect
    - Not efficient (could call clients in parallel)
    + Simple

Hidden latency


trait ProductClient {						
  def getProduct(productId: Long): Product
}

// blocked until product is retrieved from http service
val product: Product = productClient.getProduct(productId)

Explicit latency


trait ProductClient {						
  def getProduct(productId: Long): Future[Product]
}

// Future[Product] is returned immediately
val productF: Future[Product] = productClient.getProduct(productId)

Future is like a box that at some point of future might contain a value or error inside.
